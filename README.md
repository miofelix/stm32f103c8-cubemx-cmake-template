# stm32f103-cmake-template

Minimal CMake template for the STM32F103C8Tx (Blue Pill), generated by STM32CubeMX. Clocks are configured for 72 MHz (HSE 8 MHz + PLL x9). Builds with Ninja + `arm-none-eabi-gcc` and flashes via OpenOCD (ST-LINK / DAPLink).

## Project Layout

```text
.
├── Core/                         # Application sources generated by STM32CubeMX
├── Drivers/                      # STM32 HAL and CMSIS drivers
├── cmake/
│   ├── firmware-images.cmake     # Generate .hex and .bin after linking
│   ├── flash-targets.cmake       # Add flash, erase and reset targets
│   ├── openocd-flash.cmake       # OpenOCD command wrapper
│   └── stm32cubemx/              # STM32CubeMX generated CMake target
├── docs/
│   └── STM32F103C8T6-CubeMX-Configuration-Guide.md  # CubeMX setup reference
├── scripts/
│   └── rename.sh                 # Rename the project (CMake target, .ioc, docs)
├── CMakeLists.txt
├── CMakePresets.json
├── STM32F103XX_FLASH.ld
├── startup_stm32f103xb.s
└── stm32f103_project.ioc
```

## Use as a template

After copying this repo, rename the project in one step — it updates the CMake
target (`CMAKE_PROJECT_NAME`), the CubeMX `.ioc` (fields **and** filename), the
VS Code debug config, and the docs:

```sh
scripts/rename.sh my_project      # letters, digits, '_' and '-'
rm -rf build && cmake --preset Debug
```

The script reads the current name from `CMakeLists.txt`, so it is safe to re-run.
Targeting a different STM32 is a separate manual step: swap `startup_*.s`, the
`*.ld` linker script, and the `STM32F103xB` / `-mcpu=cortex-m3` settings.

## Documentation

- [STM32CubeMX Configuration Guide](docs/STM32F103C8T6-CubeMX-Configuration-Guide.md) — a complete reference for configuring the STM32F103C8T6 in CubeMX: clock tree, full LQFP48 pinout, peripheral setup (ADC/USART/I2C/SPI/TIM/USB/CAN/DMA/NVIC), and pin-conflict avoidance.

## Prerequisites (macOS)

```sh
brew install cmake ninja open-ocd
brew install --cask gcc-arm-embedded
```

Verify the tools are on `PATH`:

```sh
cmake --version
ninja --version
openocd --version
arm-none-eabi-gcc --version
```

If CMake cannot find the ARM toolchain, confirm that Homebrew linked the binaries into `/opt/homebrew/bin`:

```sh
ls /opt/homebrew/bin/arm-none-eabi-gcc
```

## Build

Debug build:

```sh
cmake --preset Debug
cmake --build --preset Debug
```

Release build:

```sh
cmake --preset Release
cmake --build --preset Release
```

Build outputs are generated under `build/<Preset>/`, for example:

```text
build/Debug/stm32f103_project.elf
build/Debug/stm32f103_project.hex
build/Debug/stm32f103_project.bin
build/Debug/stm32f103_project.map
```

## Flash

The default flash adapter is ST-LINK:

```sh
cmake --build --preset Debug --target flash
```

Use DAPLink/CMSIS-DAP for the current command:

```sh
FLASH_ADAPTER=daplink cmake --build --preset Debug --target flash
```

Other OpenOCD targets:

```sh
cmake --build --preset Debug --target erase
cmake --build --preset Debug --target reset
```

The default OpenOCD target file is `target/stm32f1x.cfg`. Override it only when the board or chip variant requires another target script:

```sh
OPENOCD_TARGET=target/stm32f1x.cfg cmake --build --preset Debug --target flash
```

## OpenOCD Checks

Check ST-LINK connection:

```sh
openocd -f interface/stlink.cfg -f target/stm32f1x.cfg
```

Check DAPLink/CMSIS-DAP connection:

```sh
openocd -f interface/cmsis-dap.cfg -f target/stm32f1x.cfg
```

Stop OpenOCD with `Ctrl-C` after the connection test succeeds.
