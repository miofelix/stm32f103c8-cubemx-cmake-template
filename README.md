# stm32f103c8-cubemx-cmake-template

Minimal macOS-only CMake template for the STM32F103C8Tx (Blue Pill), generated by STM32CubeMX. Clocks are configured for 72 MHz (HSE 8 MHz + PLL x9). Builds with Ninja + `arm-none-eabi-gcc` from Homebrew and flashes via OpenOCD (ST-LINK / DAPLink).

## Use as a template

After copying this repo, rename the project in one step ‚Äî it updates the CMake
target (`CMAKE_PROJECT_NAME`), the CubeMX `.ioc` (fields **and** filename), the
VS Code debug config, and the docs:

```sh
scripts/rename.sh my_project      # letters, digits, '_' and '-'
rm -rf build && cmake --preset Debug
```

The script reads the current name from `CMakeLists.txt`, so it is safe to re-run.
Targeting a different STM32 is a separate manual step: swap `startup_*.s`, the
`*.ld` linker script, and the `STM32F103xB` / `-mcpu=cortex-m3` settings.

## Ownership model

STM32CubeMX owns the generated firmware code and its dependency closure:

- `Core/`
- `Drivers/`
- `cmake/stm32cubemx/`
- `startup_*.s`
- `*.ld`
- the generated parts of the root `CMakeLists.txt`

The tracked `.ioc` file is the source of truth for clocks, pins, peripherals,
middleware, HAL modules, and generated source selection. Add or remove a
peripheral in CubeMX, then regenerate the project. Do not manually vendor the
complete HAL package or override the generated `STM32_Drivers` source list.

The template owns the surrounding assets: presets, root helper CMake modules,
scripts, CI, editor configuration, and documentation. CubeMX's `.mxproject` is
local workspace metadata and is intentionally ignored.

After every CubeMX regeneration, run:

```sh
scripts/check-template.sh
cmake --workflow --preset Debug
cmake --workflow --preset Release
```

## Prerequisites (macOS only)

This template is intentionally maintained for macOS + Homebrew only. Linux and
Windows are not supported targets for this repository or its CI.

CMake 3.25 or newer is required because `CMakePresets.json` uses preset schema
version 6 and workflow presets.

```sh
brew install cmake ninja open-ocd
brew install --cask gcc-arm-embedded
```

Verify the tools are on `PATH`:

```sh
cmake --version
ninja --version
openocd --version
arm-none-eabi-gcc --version
```

If CMake cannot find the ARM toolchain, confirm that Homebrew linked the binaries into `/opt/homebrew/bin`:

```sh
ls /opt/homebrew/bin/arm-none-eabi-gcc
```

## Build

Debug build:

```sh
cmake --preset Debug
cmake --build --preset Debug
```

One-step configure + build:

```sh
cmake --workflow --preset Debug
```

Release build:

```sh
cmake --preset Release
cmake --build --preset Release
```

Build outputs are generated under `build/<Preset>/`, for example:

```text
build/Debug/stm32f103_project.elf
build/Debug/stm32f103_project.hex
build/Debug/stm32f103_project.bin
build/Debug/stm32f103_project.map
```

## Template CI

GitHub Actions runs on macOS, builds both Debug and Release presets, uploads the
firmware images, validates the CubeMX/template ownership contract, and checks
`scripts/rename.sh` by building a renamed temporary copy of the template.

## HAL project convention

`Core/Inc/stm32f1xx_hal_conf.h`, the HAL files under `Drivers/`, and the
`STM32_Drivers` source list are all generated from the `.ioc` configuration.
Treat them as one CubeMX-managed unit. To use another HAL peripheral, enable and
configure it in CubeMX, then regenerate so its module define, headers, sources,
clock setup, and MSP code stay consistent.

Application code should live in `USER CODE BEGIN` / `USER CODE END` blocks or in
new user-owned source files registered from the root `CMakeLists.txt`.

## Flash

The default flash adapter is ST-LINK:

```sh
cmake --build --preset Debug --target flash
```

Use DAPLink/CMSIS-DAP for the current command:

```sh
FLASH_ADAPTER=daplink cmake --build --preset Debug --target flash
```

Other OpenOCD targets:

```sh
cmake --build --preset Debug --target erase
cmake --build --preset Debug --target reset
```

The default OpenOCD target file is `target/stm32f1x.cfg`. Override it only when the board or chip variant requires another target script:

```sh
OPENOCD_TARGET=target/stm32f1x.cfg cmake --build --preset Debug --target flash
```

## OpenOCD Checks

Check ST-LINK connection:

```sh
openocd -f interface/stlink.cfg -f target/stm32f1x.cfg
```

Check DAPLink/CMSIS-DAP connection:

```sh
openocd -f interface/cmsis-dap.cfg -f target/stm32f1x.cfg
```

Stop OpenOCD with `Ctrl-C` after the connection test succeeds.
